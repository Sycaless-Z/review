# 阅读笔记

## Amortized Analysis 摊还分析

goal: 证明算法的通道利用率为 $1/e - O(\epsilon)$ $\leftarrow$ 分析势函数 (potential function) $\Phi$ 的期望变化

$\Phi_{t}$: 时隙 $t$ 结束后的势能

$n_{t} \ge 0$: 时隙 $t$ 开始前注入系统的数据包数量

要证: 
$$\mathbb{E}[\Phi_{t} - \Phi_{t-1}] \le -(1 - O(\epsilon)) + (e + O(\epsilon)) \cdot n_{t}$$

Initialization Rule 初始化规则: 
激活时，设备 $i$ 设置 $p_{i} \leftarrow \epsilon^{2}$

直观理解: 新数据包带来约 $e$ 单位势能; 每个时隙算法期望消耗约 1 单位势能 \
-> 像摊还分析里的 "记账": 低效时隙没有白白浪费, 它让系统状态往更好方向移动

### 3.1 The Potential Function

$$\Phi = A(n) + B(\gamma) + C$$

三个部分:
- $A(n)$: 只看还没传完的数据包数量
- $B(\gamma)$: 看当前总冲突度是否偏离最优值 $c=1$
- $C$: 处理某个 $p_i$ 特别大/特别小时带来的不均衡问题

主项: $A(n) = en$ \
如果 $\gamma$ 在 $[-\sqrt{\epsilon}, \sqrt{\epsilon}]$ 内, 成功概率接近 $1/e$ \
-> $A$ 的期望下降约为 $e\cdot 1/e = 1$

辅助项 $B(\gamma)$: 让 $\gamma$ 离 0 越远势能越高

$$B^{\prime}(\gamma)=\begin{cases}-\frac{5}{3\epsilon}&\gamma<-1\\\frac{5}{3\epsilon}\gamma&\gamma\in[-1,1]\\\frac{5}{3\epsilon}&\gamma>1\end{cases}$$

即 $\gamma\in[-1,1]$ 时 $B(\gamma)=\frac{5}{6\epsilon}\gamma^2$ \
作用: 当冲突度太小/太大时, 更新规则会让 $\gamma$ 往 0 靠, 从而使 $B$ 下降, 用来支付低效率时隙

第三项:
$$C=\frac{5}{3\epsilon}(\gamma-\ln p_{min})$$

$p_{min}=\min_i p_i$ \
更新规则会让所有 $p_i$ 同乘一个系数, 所以 $\gamma$ 和 $\ln p_{min}$ 同时变化, $C$ 不变 \
作用: 当某个设备成功传输并退出时, 防止 $B$ 因 $\gamma$ 突降而增加太多

### 3.2 Analysis

分析一个时隙分三步:
1. 新数据包注入导致 $\Phi$ 增加
2. 传输 + 更新导致 $A+B$ 的期望变化
3. 若成功传输, 删除该设备的 $p_i$ 后导致 $B+C$ 的变化

假设 (*): 
$$p_{min}\le\epsilon^2$$

Lemma 1: 插入 $m$ 个数据包, $\Phi$ 增加 $(e+O(\epsilon))m$ \
因为 $A$ 每个包增加 $e$, $B,C$ 的额外变化可以控制在 $O(\epsilon)$

Lemma 2:
- 每个时间步中, $A+B$ 的期望变化为 $-1+O(\epsilon)$
- 成功传输时, $B+C$ 的最坏变化至多为 0

证明思路分三类看 $\gamma$:
- $\gamma$ 远离 0: $B$ 在线性部分, 更新规则的吸引力足够让 $B$ 下降约 1
- $\gamma\in[-1,1]$: $B$ 是二次函数, 结合 $p_{suc}(\gamma)>1/e-\gamma^2/4$, 得到 $A+B$ 仍下降 $1-O(\epsilon)$
- $\gamma$ 在边界附近: 损失只有 $O(\epsilon)$, 可由成功传输带来的 $A$ 下降补足

关键感觉: \
$A$ 负责 "真的传成功了"; $B$ 负责 "虽然没成功, 但冲突度被调好了"; $C$ 负责 "有设备退出时不要让账乱掉"

### 3.3 Variants and Extensions

Jamming 干扰:
- 对手可以制造噪音
- 一次干扰会让 $\gamma$ 减少 $\epsilon/(e-2)$
- $B$ 最多增加 $<2.33$

所以每个被干扰时隙收取约 $3.33$ 的额外开销 \
-> 去掉 $O(J)$ 个被干扰/浪费时隙后, 原效率仍可保持

Simpler Transmission Rule:
- 原规则: 设备可能主动 "制造噪音", 这是为了让分析变成精确泊松分布
- 简化规则: 以 $e^{-p_i}$ 静默, 以 $1-e^{-p_i}$ 直接传输

Lemma 3: 简化规则不会比原规则差 \
因为在唯一有差异的情况中, "传输并成功" 比 "制造噪音" 让势能下降更多

### 3.4 Channel Utilization

问题: 当活跃数据包很少时, 假设 (*) 不一定成立, 势函数表现不好 \
解决: 只要求长期利用率好; 如果系统经常只剩很少包, 本身就说明协议已经快把包清完了

$\lambda$-adversary:
- 对手可插入数据包、干扰时隙
- 但长期平均注入率受 $\lambda$ 约束

Theorem 5: 若
$$\lambda+\epsilon<\frac{1-\hat{\epsilon}}{e}$$
则无限多个时隙中, 活跃设备数会小于 $\epsilon^{-3}$ \
即长期看系统不会一直积压很多包, 通道利用率接近 $1/e$

证明工具: epoch 分段 + Azuma 不等式 \
个人理解: 如果某个阶段一直有很多包, 那势能应持续下降; 长时间不结束的概率很小

Theorem 6: 每个设备平均传输尝试次数为 $e+O(\epsilon)$ \
说明不仅时间利用率好, 设备自己的尝试次数也保持常数级
