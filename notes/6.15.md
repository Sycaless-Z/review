# 阅读笔记

## Related Work

本文思想来源: multiplicative weight update (MWU) 元算法 \
在 backoff-type protocols 中, 根据 channel feedback 更新参数本来常见; 本文主要贡献是把 MWU 型冲突消解协议做了严谨、数值精确的分析

已有退避协议变体:
- 指数增加 / 指数减少
- 乘法增加 / 线性减少
- 加法增加 / 乘法减少
- 线性或乘法增加 / 线性减少的混合

理论上, Awerbuch 等 [3] 也用过类似 MWU 的更新规则处理 jammer 模型 \
区别: 他们得到的是较小的常数效率; 本文目标是接近 $1/e$

### BEB 的稳定性

Binary Exponential Backoff (BEB) 被广泛研究

Aldous [1]: 任意常数泊松注入率 $\lambda>0$ 下, BEB 都不稳定

Bender 等 [4]:
- 注入率 $\Omega(\log\log n/\log n)$ 下不稳定
- 注入率 $O(1/\log n)$ 下稳定

理解: BEB 工程上简单, 但理论上长期高负载时会积压/失稳

BEB 不稳定 $\rightarrow$ 发展出更复杂的稳定协议 [6,3,7] \
这些协议和本文一样, 需要 channel feedback 能区分:
- silence
- noise

### $1/e$ 阈值

本文协议对 stateless algorithms 来说达到最优阈值 $1/e$

stateless: 每个设备每个时间步执行相同算法, 不额外记复杂状态

但若假设数据包到达服从泊松分布, $1/e$ 不是绝对上限:
- Mosely & Humblet [25], Tsybakov & Mikhailov [29]: 稳定到约 $0.48776$
- 已知最优上界: $0.5874$ [23]

理解: \
$1/e$ 是在较弱信息/无状态限制下的最优, 不是所有模型下的最优

### Feedback 的强弱

ternary feedback: 区分 silence / success / noise

反馈强弱会明显影响上限:
- 若只有发射器收到反馈, 注入率 $>0.42$ 时无协议稳定 [13]
- 若通道能报告准确发射器数量, $n$ 个同步设备可在 $n+o(n)$ 个时隙内解决, 效率 $1-o(1)$ [26]

直观: 反馈信息越多, 协议越容易接近完美调度; 本文处在较现实的三元反馈模型中

其他工作:
- 异构数据包大小的 BEB 变体 [5]
- 泊松注入率低于 $1/e$ 时, 每包期望延迟 $O(1)$ 的协议 [14]

总结: \
Related Work 主要说明本文的位置: 它不是第一个用反馈调概率的退避协议, 但用非常简单的 MWU 更新规则, 在 stateless + ternary feedback 模型下接近 $1/e$ 最优效率, 并给出了严格分析

## Conclusions

本文结论:
- 给出一个简单自然的 contention resolution protocol
- channel utilization 可任意接近 $1/e$
- 可抵抗 jammer 干扰常数比例的时隙

$1/e$ 阈值:
- 对 stateless algorithms 是最优的
- 若不显著增加算法复杂度, 很难继续提高效率

作者猜想: \
对泊松注入率可达到 $0.48776$ 的协议 [25,29], 可能也能用类似 MWU 机制改造成适用于 adversarial injection 的协议

Transmission attempts:
- 本文: 每个设备平均尝试次数 $e+O(\epsilon)$
- [6]: $O(\log^{2}(n+J))$

本文优势: 发送尝试次数是常数级, 与 $n,J$ 无关

不足 / open problem: \
协议要求设备每个时间步都监听 channel feedback, 监听本身也消耗能量

[7] 中 energy: 访问或监听通道所花的时隙数

开放问题: \
能否同时做到
- energy cost 为 $\text{poly}(\epsilon^{-1},\log T)$
- channel utilization 为 $1/e-O(\epsilon)$

理解: \
本文主要把 "效率" 和 "发送尝试次数" 做得很好, 但还没解决低能耗监听问题
