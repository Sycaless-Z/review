# Q: contention resolution是什么

**Contention resolution（竞争解决 / 冲突解决）** 是计算机网络、操作系统、数据库和并行计算中常见的概念，指的是：

> 当多个用户、设备、进程或线程同时争夺同一资源时，系统用来决定谁先获得资源、如何避免冲突以及如何恢复冲突的机制。

“Contention” 意为**竞争、争用**，而 “resolution” 意为**解决**。

## 1. 网络中的 Contention Resolution

最经典的例子是以太网。

假设三台计算机同时向网络发送数据：

```text
PC A ──┐
        ├── Network
PC B ──┤
        │
PC C ──┘
```

如果它们同时发送，就会发生**碰撞（collision）**。

此时使用一种 contention resolution 机制：

### CSMA/CD

CSMA/CD

流程：

1. 先监听信道是否空闲
2. 空闲则发送
3. 如果检测到碰撞
4. 停止发送
5. 等待随机时间
6. 再重试

这里的“随机退避（backoff）”就是一种 contention resolution 方法。

---

## 2. 操作系统中的 Contention Resolution

多个线程争抢同一个锁：

```java
Thread A -> lock
Thread B -> lock
Thread C -> lock
```

只有一个线程能获得锁。

解决方式包括：

* Mutex（互斥锁）
* Semaphore（信号量）
* Spinlock（自旋锁）
* Priority Scheduling（优先级调度）

例如：

```text
A获得锁
B等待
C等待
```

当 A 释放锁后：

```text
B获得锁
```

这就是 contention resolution。

---

## 3. 数据库中的 Contention Resolution

多个事务同时修改同一条记录：

```sql
UPDATE account
SET balance = balance - 100
```

和

```sql
UPDATE account
SET balance = balance + 50
```

如果同时执行，可能导致数据不一致。

数据库采用：

* Row Lock（行锁）
* MVCC（多版本并发控制）
* Timestamp Ordering（时间戳排序）
* Deadlock Detection（死锁检测）

来解决资源争用。

例如在 PostgreSQL 或 MySQL 中，锁管理器就是 contention resolution 的重要组成部分。

---

## 4. 无线通信中的 Contention Resolution

在 Wi-Fi 中，多台设备共享无线信道：

```text
Phone
Laptop
Tablet
TV
```

都想发送数据。

Wi-Fi 使用：

CSMA/CA

特点：

* 尽量避免碰撞
* 发送前等待随机时间
* 必要时重传

这也是 contention resolution。

---

## 5. 更抽象的理解

可以把 contention resolution 理解成：

```text
多个参与者
      ↓
争抢同一个资源
      ↓
发生 contention
      ↓
系统决定谁先用
      ↓
这套规则就是 contention resolution
```

资源可以是：

* 网络带宽
* CPU
* 内存
* 锁
* 数据库记录
* 无线信道
* 文件

---

## 一句话总结

**Contention resolution 就是“当多个实体同时争用同一资源时，系统用来仲裁、协调和消除冲突的机制”。** 在不同领域里，它可能表现为网络退避算法、锁机制、事务控制、调度策略等。

# Q: 翻译该论文摘要部分

### 摘要翻译 (Abstract)

我们考虑经典的冲突消解（contention resolution）问题，即多个设备协作共享某些公共资源，每个设备都需要对其进行临时的独占访问 。

为了使讨论更加具体，假设（相同的）设备在不同的时间唤醒，并且必须通过共享的多路访问通道（multiple-access channel）发送单个数据包 。在每个时间步中，它们可能会尝试发送数据包：它们会从通道接收到三元反馈 $\{0,1,2^{+}\}$，$0$ 表示静默（没有设备尝试传输），$1$ 表示成功（一个设备成功传输），$2^{+}$ 表示噪音 。

我们证明，对于任何 $\epsilon>0$，只需一个简单的策略即可实现 $1/e-O(\epsilon)$ 的通道利用率 。在每一步中，设备 $i$ 尝试以概率 $p_{i}$ 发送其数据包，然后对 $p_{i}$ 应用一个基础的乘法权重式（multiplicative weight-type）更新 ：

* 
**当听到静默 ($0$) 时**：概率更新为 $p_{i} \cdot e^{\epsilon}$ 。


* 
**当听到成功 ($1$) 时**：概率保持为 $p_{i}$ 。


* 
**当听到噪音 ($2^{+}$) 时**：概率更新为 $p_{i} \cdot e^{-\epsilon/(e-2)}$ 。



即使设备或数据包的引入具有对抗性，甚至在对手可以随意干扰时间隙（制造噪音）的情况下，该方案也能发挥良好的作用 。我们证明，如果对手干扰了 $J$ 个时间隙，那么在排除 $O(J)$ 个被浪费的时间隙后，该方案仍将实现 $1/e-\epsilon$ 的通道利用率 。

学界之前已经取得了类似的结果（Bender, Fineman, Gilbert, Young, SODA 2016），但其常数效率较低（小于 0.05），且算法更为复杂 。

# Q: 继续翻译introduction部分

### 1. 引言 (Introduction)

假设 $n$ 个相同的设备拥有想要通过共享的多路访问通道进行传输的数据包 。为简便起见，我们假设时间被划分为离散的时间隙，并且这些设备是同步的 。在每个时间隙中，它们决定是尝试发送自己的数据包还是保持空闲状态 。为了成功发送数据，设备必须在一个时间隙内独占通道：如果两个或更多设备进行传输，则会产生噪音；如果零个设备传输，则保持静默 。我们假设在每个时间步之后，所有设备都会从通道接收到指示有多少设备尝试传输其数据包的三元反馈 $\{0,1,2^{+}\}$ 。读者应当记住，我们在此考虑的是抽象的冲突消解（contention resolution）问题 。使用“数据包”、“通道”、“噪音”等术语仅仅是为了在脑海中保持一个易于具象化的问题实例 。

解决这个冲突消解问题的传统方法是通过指数退避（exponential back-off）[22] 。每个设备 $i$ 持有一个被初始化为某个常数（例如 1/2）的参数 $p_{i}$ 。在每个时间步中，它会执行以下协议：

**二进制指数退避 (Binary Exponential Backoff)：** 

* 设备 $i$ 以 $1-p_{i}$ 的概率保持静默 。


* 设备 $i$ 以 $p_{i}$ 的概率进行传输；如果不成功，则将参数设置并更新为 $p_{i}\leftarrow p_{i}/2$ 。



尽管二进制指数退避在许多应用中经验上很有用 [22, 20, 18, 27, 19, 24]，但它也存在许多缺点 。即使数据包是根据低期望值 $\lambda>0$ 的泊松分布（即一种合理的且非对抗性的输入分布）注入系统的，二进制指数退避最终也会陷入死锁，导致再也没有数据包能被成功发送 [1, 4] 。当所有 $n$ 个数据包被同时注入时，二进制指数退避需要 $n \log n$ 步才能将它们全部传输，并且每个设备都会尝试传输其数据包 $\Theta(\log n)$ 次 [4]，而在这种情况下最优的表现应是 $O(n)$ 步和 $O(1)$ 次尝试 。

近期的研究试图解决指数退避的所有缺陷，并且在许多指标上，这些研究已经取得了相当大的成功 。Bender 等人 [4] 研究了当所有 $n$ 个数据包同时到达时退避型协议的行为，并证明对于单调协议（$p_{i}$ 随时间递减），所需且充分的时间为 $O(n \log \log n / \log \log \log n)$，而对于非单调协议，则有可能在 $O(n)$ 时间内完成 。在干扰者（jammer）可以随意干扰时间隙的情况下，即使对手控制着新数据包的注入速率，在未被干扰的时间隙上实现一个（较小的）恒定吞吐量也是可能的 [6, 3] 。在文献 [6] 的协议中，每个设备平均进行 $O(\log^{2}(n+J))$ 次传输，其中 $J$ 是被干扰的时间隙数量 。Bender、Kopelowitz、Pettie 和 Young [7] 考虑了一个由电池供电设备启发的模型，其中传输和监听通道均需消耗一个单位的能量 。他们证明，当对手控制数据包的插入（但无法干扰）时，每个设备消耗 $O(\log(\log^{*} n))$ 的能量即可实现恒定的通道利用率 。这个 $O(\log(\log^{*} n))$ 的界限后来被证明是最优的 [10] 。

不幸的是，这些近期的进展在极简主义和优雅程度上远不及二进制指数退避 。在这项工作中，我们设计了一种冲突消解协议，该协议不仅匹配并显著改进了文献 [6] 的主要结果，同时还实现了接近二进制指数退避的简单性 。与退避算法类似，我们的算法只保留一个单一的数值参数（$p_{i}$），在其他方面则是无状态的：它不保存任何关于其先前动作或通道历史的信息 。
